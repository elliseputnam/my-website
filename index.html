Earthquake Prediction

Introduction to Data Science | Spring 2023

Project Description

For this project, I analyzed earthquake data from the past 50 years to uncover patterns in seismic activity and predict fault line locations. Using earthquake records from the USGS Earthquake Catalog and tectonic plate data from Kaggle, I explored how fault lines influence the distribution of seismic activity across the Americas. The project involved cleaning, transforming, and modeling the data using machine learning techniques to identify earthquake-prone regions and predict fault line locations.

Key Concepts

Earthquake Clusters and Fault Lines:
  • The model identified 12 distinct clusters of seismic activity, mainly along the western coasts of North, Central, and South America. These areas correspond closely with known tectonic plate boundaries, including the Pacific Ring of Fire, which is famous for frequent seismic activity.
  • Clusters of seismic activity often showed stronger earthquake magnitudes in the coastal regions, reflecting high seismic potential along major fault lines.
Prediction of Fault Line Locations:
  • Using Gaussian Mixture Models (GMM), I successfully predicted fault lines that closely align with known geological structures, such as the San Andreas Fault and the Cascadia Subduction Zone.
  • By modeling earthquake data with GMM, I was able to classify earthquake-prone regions, helping us understand where future seismic activity is likely to occur.
Seismic Hotspots:
  • The western coasts of the Americas emerged as the most seismically active regions, with higher earthquake magnitudes and frequencies along these fault lines. The data highlighted regions in California, Alaska, and Chile as major seismic hotspots.
  • Understanding these hotspots provides crucial information for preparedness, helping governments and organizations plan for potential disasters.
Challenges with Continuous Fault Lines:
  • Defining an accuracy metric for continuous fault lines proved difficult, as traditional evaluation methods such as accuracy or F1 scores don’t apply directly to continuous geographic data.
  • Instead, the accuracy of the model was evaluated by comparing the predicted fault line locations to known fault lines using geospatial distance metrics, such as the Root Mean Squared Error (RMSE).

Skills

  • Data Preprocessing & Transformation: Cleaned and transformed seismic data, handling missing values, filtering relevant features, and removing unnecessary data.
  • Unsupervised Learning: Applied Gaussian Mixture Models (GMM) to identify clusters of seismic activity and predict fault lines.
  • Geospatial Data Visualization: Used GeoPandas and Matplotlib to visualize earthquake data, including heatmaps and geospatial distributions.
  • Machine Learning Workflow: Implemented an end-to-end workflow from data collection to model evaluation, experimenting with different GMM variants to improve accuracy.

Movie Review Sentiment Classifier

Advanced Machine Learning

Fall 2024

Project Description

This project involves developing a sentiment classifier for movie reviews using machine learning. The classifier determines whether a review is positive or negative by leveraging a Naive Bayes model with a bag-of-words feature extraction technique. The model is integrated into a graphical user interface (GUI) for real-time predictions, allowing users to input a movie review and receive an instant sentiment classification along with a confidence score.

Key Concepts

Sentiment Classification:
  • The model classifies movie reviews as either positive or negative using a Naive Bayes classifier trained on 50,000 labeled IMDB reviews.
  • The sentiment classification helps users quickly gauge the overall opinion expressed in a review without manually reading it.
Bag-of-Words Model:
  • Text is converted into numerical features by tokenizing words and counting their frequency in the dataset.
  • Only the 10,000 most frequent words are retained, reducing dimensionality while maintaining relevant information.
Naive Bayes Classifier:
  • Multinomial Naive Bayes is used due to its efficiency and effectiveness in text classification tasks.
  • The model was trained using an 80/20 train-test split and achieved an accuracy of 82.25%.
Performance Metrics:
  • Model evaluation included accuracy (82.25%), precision (83.15%), recall (79.27%), and F1-score (81.17%).
  • A confusion matrix was generated to analyze misclassification trends, highlighting slightly higher false negatives.
GUI for Real-Time Predictions:
  • A Tkinter-based GUI allows users to input a movie review and receive a classification result.
  • The GUI also displays the model’s confidence score, providing insights into the reliability of the prediction.
Limitations and Future Improvements:
  • The model does not capture contextual relationships between words due to the bag-of-words approach.
  • Future enhancements could involve using deep learning models like transformers for improved semantic understanding.

Skills

  • Text Preprocessing: Tokenization, stopword removal, and frequency-based feature selection.
  • Machine Learning: Implemented a Naive Bayes classifier for sentiment analysis.
  • Data Visualization: Used confusion matrices and performance metrics for model evaluation.
  • GUI Development: Built an interactive interface using Tkinter for real-time sentiment analysis.
